# 匿名聊天网站 设计文档

**日期：** 2026-04-05  
**状态：** 已确认  
**技术栈：** Next.js + Supabase + Vercel + GitHub

---

## 一、项目概述

一个高度自由的匿名聊天网站。用户无需注册，进入即可聊天。首页展示所有聊天室的卡片列表，按活跃度排序。任何人都可以创建聊天室。

---

## 二、视觉风格

- **整体风格：** 暗黑极客风（Cyberpunk）
- **背景色：** 深色（`#0f0f1a`）
- **主色调：** 霓虹紫（`#a78bfa`）+ 荧光绿（`#10b981`）
- **卡片背景：** 深蓝黑（`#1a1a2e`），带微弱边框（`#2d2d4e`）
- **字体：** 系统无衬线字体，推荐 Inter
- **默认语言：** 英语（UI 文案全部英文，如 "Create Room"、"Online"、"Enter Chat" 等）
- **响应式设计：** 移动端优先（Mobile First），同时完整适配桌面端

---

## 三、首页设计

### 布局（桌面端）
- 顶部导航栏：网站 Logo、搜索框（按房间名搜索）、右上角用户头像+昵称
- 主体：聊天室宽条列表，按在线人数（活跃度）降序排列，内容区最大宽度 800px 居中
- 顶部有"+ Create Room"按钮

### 布局（移动端，手机浏览器）
- 顶部导航栏简化：Logo 居左，头像图标居右，隐藏搜索框（改为图标按钮点击展开）
- 聊天室卡片全宽显示，上下滚动浏览
- 底部固定悬浮按钮"＋"用于创建聊天室，方便拇指操作
- 聊天室内部布局：
  - 顶部显示房间名 + 在线人数
  - 在线用户列表默认隐藏，点击右上角图标展开（抽屉式侧边栏）
  - 消息输入框固定在屏幕底部，键盘弹出时页面自动上移
  - 消息气泡适配小屏，自己发的消息靠右，他人消息靠左

### 聊天室卡片（列表项）
每条显示：
- 房间名称（带 emoji 可选）
- 房间简介（一行文字）
- 当前在线人数（绿点 + 数字）
- 热门标签（在线人数 ≥ 50 时显示"🔥 热门"标签）

### 排序规则
按当前在线人数实时降序排列。无人在线的房间排在末尾。

## 四、用户系统

### 匿名进入
- 用户首次访问，自动生成：
  - 随机昵称（如"神秘狐狸"、"夜行者2847"等）
  - 随机头像（从预设的 emoji 头像池中随机取一个）
- 用户信息存储在 Supabase，同时在浏览器 `localStorage` 中保存用户 token
- 刷新页面后，凭 token 恢复身份（在 token 有效期内）

### 可选绑定
- 右上角点击头像/昵称，可进入个人资料页
- 支持绑定：**邮箱**（Supabase 邮件验证）、**手机号**（Supabase SMS 验证）
- 绑定后，用户可以在新设备/清除浏览器后通过邮箱/手机找回身份
- 绑定不影响匿名性（昵称和头像仍然是随机的，不显示真实信息）

### 数据保留规则
- **未绑定用户：** 账号数据保留 **5 天**，到期自动删除。该用户历史消息中的发送者显示为"已注销用户"，消息内容本身依照消息保留策略（30天）继续保留
- **已绑定用户：** 账号数据永久保留（直到用户主动删除）

## 五、聊天室

### 创建
- 任何用户（包括匿名用户）均可创建聊天室
- 创建表单：房间名称（必填，最多 30 字）、房间简介（选填，最多 100 字）
- 创建后立即出现在首页列表

### 进入聊天室
- 点击首页卡片直接进入，无需额外确认步骤
- 进入后可见：实时消息流、在线用户列表（侧边栏）、消息输入框

### 房间管理权限
| 权限 | 普通用户 | 管理员 | 创建者 |
|------|---------|--------|--------|
| 发送消息 | ✅ | ✅ | ✅ |
| 任命管理员 | ❌ | ❌ | ✅ |
| 禁言用户 | ❌ | ✅ | ✅ |
| 踢出用户 | ❌ | ✅ | ✅ |
| 解散房间 | ❌ | ❌ | ✅ |

### 禁言规则
- 被禁言用户可继续查看消息，但无法发送消息
- 禁言可设置时长（5分钟 / 1小时 / 永久）

## 六、消息系统

### 实时性
- 使用 Supabase Realtime（WebSocket）广播消息，实现毫秒级实时同步

### 在线人数实时机制

**房间内部（精确计数）：**
- 用户进入聊天室页面 → 加入该房间的 Supabase Presence 频道
- 用户关闭页面/跳走 → Presence 自动检测断线，将其移除
- Presence 事件（join/leave）触发客户端回调，同步更新 `rooms.online_count`

**首页卡片（实时展示）：**
- 首页订阅 Supabase Realtime 对 `rooms` 表的数据库变更（INSERT/UPDATE）
- 一个 WebSocket 连接监听所有房间的 `online_count` 字段变化
- 任意房间人数变动时，首页对应卡片的数字立即刷新，无需手动刷新页面

**断线保障：**
- 若用户因网络异常断开，Supabase Presence 会在超时后（约 10 秒）自动触发 leave 事件，确保人数不会虚高

### 消息结构
每条消息包含：
- 消息内容（纯文本，初期不支持图片/文件）
- 发送者昵称 + 头像
- 发送时间
- 房间 ID

### 保留策略
- **初期（流量小）：** 消息保留 **30 天**，进入房间可看到历史记录
- **成熟期（流量大后）：** 调整为保留 **24 小时**
- 使用 Supabase `pg_cron` 定时任务每天自动清理过期消息

## 七、技术架构

```
用户浏览器
    │
    ▼
Vercel（托管）
    │
    ▼
Next.js 应用
    ├── 页面路由（App Router）
    │     ├── / （首页，聊天室列表）
    │     ├── /room/[id] （聊天室页面）
    │     └── /profile （个人资料/绑定页面）
    │
    └── Supabase Client
          ├── Auth（匿名登录 + 邮箱/手机绑定）
          ├── Database（PostgreSQL）
          │     ├── users 表
          │     ├── rooms 表
          │     ├── messages 表
          │     ├── room_members 表（记录权限）
          │     └── mutes 表（禁言记录）
          └── Realtime（WebSocket 消息广播）
```

### 数据库表结构（简版）

**users**
- `id` UUID（Supabase Auth 生成）
- `nickname` text
- `avatar_emoji` text
- `is_bound` boolean
- `created_at` timestamp
- `last_seen_at` timestamp

**rooms**
- `id` UUID
- `name` text
- `description` text
- `creator_id` UUID（外键 users.id）
- `created_at` timestamp
- `online_count` int（持久化存储，由 Presence 事件驱动实时更新，见下方在线人数机制）

**messages**
- `id` UUID
- `room_id` UUID（外键 rooms.id）
- `user_id` UUID（外键 users.id）
- `content` text
- `created_at` timestamp
- `expires_at` timestamp（= created_at + 30天）

**room_members**
- `room_id` UUID
- `user_id` UUID
- `role` enum（`member` / `admin` / `creator`）

**mutes**
- `room_id` UUID
- `user_id` UUID
- `muted_by` UUID
- `expires_at` timestamp（null = 永久）

## 八、部署流程

1. 在 GitHub 创建仓库
2. 在 Supabase 创建项目，配置数据库表和 Auth
3. 在 Vercel 导入 GitHub 仓库，填写 Supabase 环境变量
4. 推送代码到 GitHub → Vercel 自动部署
5. （可选）在 GoDaddy 购买域名，绑定到 Vercel
